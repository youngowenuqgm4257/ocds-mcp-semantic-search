## Overview

OCDS-MCP is designed for procurement discovery where plain keyword search is not enough. By combining OCDS-aware tender processing with semantic matching, it can surface relevant opportunities even when notices use different phrasing, come from different languages, or originate from different sources.

It fits use cases where procurement teams, analysts, and AI assistants need to evaluate tenders against company capabilities, maintain private company profiles, or process many records at once. With a web interface and MCP-compatible output, it slots into research and assistant-driven workflows without forcing a narrow usage pattern.

## Usage

A typical flow is straightforward: search, inspect, compare, and integrate.

1. Enter tender terms, procurement details, or company information.
2. Review semantic matches rather than relying only on exact text matches.
3. Compare tender results against private company profiles.
4. Use German-English support when working across multilingual records.
5. Export or consume MCP-compatible JSON when integrating with an AI assistant.

Example workflow:
- Search for a German procurement notice
- Refine results using tender matching
- Compare shortlisted items with company capabilities
- Process larger sets through batch mode when needed
